# GTM Commercial Underwriter Co-Pilot

A high-precision agentic assistant designed to automate the ingestion and triaging of complex commercial risk submissions. This solution orchestrates a multi-agent workflow that extracts data from unstructured loss runs and COIs, validates them against underwriting guidelines, and prepares a ready-to-bind summary for the human underwriter.

## Solution Architecture

### Multi-Agent Orchestration

The solution employs three specialized agents coordinated by an orchestration layer:

```

         │                                │
```

### AWS Services Integration

| Service | Purpose | Agent |
|---------|---------|-------|
| **Amazon Textract** | Document analysis and OCR | Extraction |
| **Amazon S3** | Document and data storage | All Agents |
| **Amazon Bedrock** | Advanced multi-step reasoning | All Agents |
| **Amazon SageMaker** | Risk scoring model inference | Validation |
| **AWS Glue** | Data cataloging and normalization | Extraction/Validation |

| **SNS** | Notifications and alerts | All Agents |

| **CloudWatch** | Logging and monitoring | All Agents |

## Submission Workflow

### Step 1: Reception & Orchestration
- Broker emails/uploads a 50-page PDF submission
- S3 EventBridge triggers Orchestration Agent
- Workflow context created and persisted
- Extraction task queued

### Step 2: Data Extraction
- Textract extracts text, tables, and forms
- Bedrock intelligently identifies key fields:
  - Insured name and contact info
  - Policy numbers and coverage limits
  - Loss history (last 5 years)
  - Current renewal dates
  - Special conditions/exclusions
- Historical loss data cross-referenced from S3
- Extraction results validated and saved

### Step 3: Business Rule Validation
- SageMaker risk scoring model evaluates submission
- Extracted data compared against appetite guidelines:
  - Acceptable coverage limits
  - Excluded industries/occupations
  - Maximum prior loss counts
  - Minimum years in business
- Bedrock provides detailed compliance analysis
- Flagged/out-of-scope risks identified

### Step 4: Summary & Recommendation
- Comprehensive underwriting memo generated by Bedrock
- Key findings and risk summary compiled
- Final recommendation produced:
  - **APPROVE**: Low-risk, standard terms
  - **APPROVE_WITH_CONDITIONS**: Acceptable with modifications
  - **REFER_TO_UNDERWRITER**: Requires human review
- HTML report generated for easy review
- Notifications sent to underwriting team

## Deployment

### 1. Build Lambda Packages

```bash
cd lambda_functions
python3 build.py
cd ..
```

This creates deployment ZIP files in `lambda_functions/build/`

### 2. Configure Variables

Edit `terraform.tfvars` with your settings:

```hcl
aws_region           = "us-east-1"
environment          = "dev"
bedrock_model_id     = "anthropic.claude-3-sonnet-20240229-v1:0"
sagemaker_endpoint   = "your-endpoint-name"  # Optional

tags = {
  Owner       = "Underwriting-Team"
  CostCenter  = "Insurance"
}
```

### 3. Initialize and Deploy

```bash
terraform init
terraform plan
terraform apply
```

### 4. Configure S3 Event Triggers

To enable automatic submission processing:

```bash
aws s3api put-bucket-notification-configuration \
  --bucket <submission-bucket> \
  --notification-configuration '{
    "EventBridgeConfiguration": {}
  }'
```

Create EventBridge rule to trigger Orchestration Agent on S3:PutObject events.

## Configuration

### Customizing Appetite Guidelines

Create `appetite-guides/guidelines.json` in your S3 bucket:

```json
{
  "policy_types": [
    "Commercial General Liability",
    "Commercial Auto",
    "Property",
    "Workers Compensation"
  ],
  "min_coverage_limit": 500000,
  "max_coverage_limit": 5000000,
  "max_loss_ratio": 0.60,
  "max_prior_losses_3_years": 3,
  "min_years_in_business": 2,
  "excluded_industries": ["Mining", "Aviation", "Nuclear"]
}
```

### Historical Loss Data Format

For each insured, create `historical-data/<insured_name>/losses.json`:

```json
{
  "prior_losses": [
    {"year": 2023, "amount": 50000},
    {"year": 2022, "amount": 25000},
    {"year": 2021, "amount": 0}
  ],
  "loss_ratio": 0.15,
  "years_insured": 5
}
```

### Common Issues

| Issue | Cause | Solution |
|-------|-------|----------|
| Textract fails | Document not in S3 | Verify S3 path and bucket permissions |
| Bedrock timeout | Model overloaded | Increase Lambda timeout or retry |
| SageMaker error | Endpoint not active | Create/activate SageMaker endpoint |
| SNS not sending | Topic not subscribed | Create email/SMS subscriptions |

## Performance Tuning

### Lambda Configuration

Adjust in `terraform.tfvars`:

```hcl
lambda_timeout = 900      # 15 minutes for document processing
lambda_memory  = 512      # 512 MB for Lambda functions
agents_count   = 3        # Parallel agent processing
```

### SQS Batch Processing

Modify `lambda.tf` batch sizes:

```hcl
batch_size = 1  # Process one message at a time
# Increase to 10 for higher throughput, increase visibility timeout
```

## Cost Optimization

- **S3 Lifecycle Policies**: Automatic archival to Glacier after 90 days
- **Lambda Reserved Concurrency**: Prevent costs from runaway scaling
- **Bedrock Model Selection**: Claude 3 Haiku for cost-sensitive use cases
- **SageMaker Batch Transform**: For bulk risk scoring instead of real-time

## Security Best Practices

1. **IAM Policies**: Follow least-privilege principle (already configured)
2. **S3 Encryption**: Enable default encryption on submission bucket
3. **VPC Configuration**: Deploy Lambda in VPC for additional isolation
4. **Secrets Management**: Use AWS Secrets Manager for sensitive data
5. **Audit Logging**: Enable CloudTrail for compliance tracking
